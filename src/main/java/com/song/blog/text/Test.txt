Test
- 제대로 작동하는지 테스트
- 서버를 키고 끄고 DB 잘 작동하는지 등등을 최소화해줌


테스트의 단계
1. 예상 시나리오(Expectaion)
2. 실제 코드와 비교
3. 검증
- 테스트를 통과한 경우 더 좋은 코드로 Refactoring
- 테스트에 실패한 경우  Debugging으로 고쳐줌


Test Case
- 테스트는 다양한 상황에 맞춰 작성함


TDD(Test Driven Development)
- 테스트 코드를 먼저 만들고
- 이를 통과하는 최소한의 코드로 시작해서
- 점진적으로 개선, 확장해 나가는 개발 방식
